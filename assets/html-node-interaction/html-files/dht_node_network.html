DHT Network Key Discovery Simulator

Decentralized Hash Table (DHT) Network Rules:

  • Network nodes spawn continuously and form a distributed hash table
  • Each node stores a key-value pair (hash: signature key 1-1000)
  • Nodes query their neighbors to search for the target signature
  • Queries propagate through the network via node connections
  • Nodes connect to nearby peers (shown as grey lines)
  • Active queries are shown as orange connections
  • Nodes dissipate after 8 seconds (simulating churn)
  • Goal: Distributed network finds the target signature key!
